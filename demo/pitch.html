Why now

Three pressures converging.

  • The legacy reconciliation tool is end-of-life. No active vendor support. No Windows 11 hardening. UI unchanged since launch.
  • Month-end reconciliation is manual. Ops exports from the core (Flexcube / T24 / Finacle / equivalent), pulls SWIFT, eyeballs matches in Excel. [FILL: hours per cycle].
  • Audit has no breadcrumbs. "Who cleared this on the 14th?" → a spreadsheet filename.
The problem isn't the legacy tool itself — it's that legacy tool plus manual Excel stitching is how books actually close. Three risks in one: a tool you can't patch, hours you can't reclaim, decisions you can't trace.

What Kilter does

Pairs SWIFT messages with Flexcube transactions, proposes matches in tiers, and lets ops confirm or reject — with every click logged.

Six user-visible parts:

  1. Intake — drop SWIFT .out/.xml and Flexcube .xlsx, or let the auto-scanner pick them up from messages/.
  2. Review queue — ranked match candidates; confirm in one click, keyboard-first.
  3. Open items ledger — breaks roll forward across sessions until cleared or written off.
  4. Scoped dashboard — each user picks the access areas they work in (branch, cluster, or all).
  5. Export + audit — xlsx report for finance, full immutable audit log for compliance.
  6. Automation — 7 built-in scheduler jobs run overnight, unattended.

The matching engine

Four tiers, strongest first. Nothing auto-confirms without ops.

TierWhat matchesExample
1 · StrictReference + amount + date1:1, no judgement needed
2 · Ref hit, amounts differSame ref, amount off by fees / FXWire-fee deduction
3 · Amount + date, no refSame amount, same dayStatement without narrative
4 · Amount ±1 daySame amount, next-day bookingWeekend / cut-off timing

Ops decides tier 2–4. The engine proposes and ranks. Tier 1 can auto-confirm; tiers 2–4 always need a human.

The engine isn't doing the ops team's job — it's doing the searching so ops can do the deciding. Tier 1 is the "free" matches. Tier 2-4 is where judgement lives, and that's exactly the work we want humans on.

The review experience

  • One candidate at a time — no Excel scrolling.
  • Confirm / Reject / Skip — keyboard shortcuts built in (C / R / J·K to navigate).
  • Swap — if the top pick is wrong, surface the next ranked candidate without re-running the engine.
  • Stats strip at the top: pending decisions, confirmed, unmatched SWIFT, unmatched Flex — live as you work.
  • “Queue cleared → Download reconciliation xlsx” when you're done.

Access-area scoping

For the “I just need my branch” crowd.

  • Top-bar picker, present on every page.
  • Single area (most common) or multi-select (regional supervisors).
  • Scope filters Dashboard, Cash accounts, Sessions — users only see work relevant to them.
  • Areas configurable per deployment — import from your existing taxonomy, or define your own.
Ops asked: I only work on Branch 001 — don't show me 102 other areas. Done. Leadership asked: I want to see the whole bank. Leave it on All areas — same screen, different scope.

Roles & audit

Four roles

  • admin — everything, including ingest and user management.
  • ops — reconcile, confirm/reject, export.
  • audit — read-only: activity log + exports.
  • internal_control — read-only: same as audit, scoped separately.

MFA  TOTP — works with Microsoft Authenticator, Google Authenticator, Authy, or any TOTP app. Enrollment via one-time token link.

Everything is logged

Login, match decision, export, scope change, account registration — one immutable audit_log table. Exports as CSV for regulators.

The audit log is the product within the product for compliance teams. The CSV export means no one needs a database query tool to answer audit questions.
